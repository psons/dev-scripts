The directory is for specs that are code ready.

In simple cases either a use case or a spec in the parent of this directory is sufficient.
In other cases, it may be useful to combine various sources and be more vague to allow AI latitude to 
find common known patterns and libraries to drive an implementation.  Having a code ready implementation guid
can streamline review of the design for humans and find ambiguities or differing interpretations that may 
cause incorrect code.
